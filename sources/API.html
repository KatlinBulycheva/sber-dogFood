Api для проектной работы

Как сделать запрос к серверу

Адрес сервера проекта: https://api.react-learning.ru/

При каждом запросе нужно передавать токен. Вот пример запроса:

fetch('https://api.react-learning.ru/v2/group-7/posts', {
  headers: {
    authorization: 'Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJfaWQiOiI2MjJmOTk5MmFlNWM0MGMxMGMxMWRmZTQiLCJpYXQiOjE2NDcyODY2ODEsImV4cCI6MTY3ODgyMjY4MX0.WHKXAErKZtY445yXecOFZsx981MuXicJti-okSY-tac'
  }
})
  .then(res => res.json())
  .then((result) => {
    console.log(result);
  });

Обратите внимание, что при запросе нужно передавать токен в заголовке в заголовке authorization со значение Bearer ${ВАШ_JWT}. Это необходимое условие. Если не передать серверу токен или передать неправильный — сервер ответит ошибкой. Делайте это в каждом запросе.

1. Загрузка информации о пользователе с сервера

Информация о пользователе должна подгружаться с сервера. Чтобы осуществить это, сделайте GET-запрос на URL:

GET https://api.react-learning.ru/v2/:groupId/users/me

При запросе не забудьте передать токен. Если запрос прошёл успешно, в ответе вы получите объект пользователя:

{
    "name": "Максим Иванов",
    "about": "Наставник",
    "avatar": "https://u.kanobu.ru/articles/pics/7e6dc974-43f4-4ad0-9a55-2465566e9662.jpg",
    "_id": "622bd81b06c7d323b8ae4614",
    "email": "maxim_91@inbox.ru",
    "__v": 0
}

Используйте свойства name, about и avatar по своему усмотрению, например в шапке проекта. Свойство _id — это идентификатор пользователя, в данном случае вашего.

3. Редактирование профиля

Отредактированные данные профиля должны сохраняться на сервере. Для этого отправьте запрос методом PATCH:

PATCH https://api.react-learning.ru/v2/:groupId/users/me

В заголовках запроса, кроме токена, необходимо отправить Content-Type, а в теле — JSON с двумя свойствами — name и about. Значениями этих свойств должны быть обновлённые данные пользователя. Вот пример такого запроса:

fetch('https://api.react-learning.ru/posts/users/me', {
  method: 'PATCH',
  headers: {
    authorization: 'Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJfaWQiOiI2MjJmOTk5MmFlNWM0MGMxMGMxMWRmZTQiLCJpYXQiOjE2NDcyODY2ODEsImV4cCI6MTY3ODgyMjY4MX0.WHKXAErKZtY445yXecOFZsx981MuXicJti-okSY-tac',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'Василий',
    about: 'Программист'
  })
});

Если обновление прошло успешно, в теле ответа от сервера вы получите обновлённые данные пользователя:

{
    "name": "Василий",
    "about": "Программист",

Чтобы добавить на сервер новый пост, отправьте POST-запрос:

POST https://api.react-learning.ru/v2/:groupId/posts

В заголовках запроса, кроме токена, необходимо отправить Content-Type, а в теле — JSON с свойствами:

{

  "tags": ["legendary", "peace", "kaif"] //тип данных массив строк

В заголовках запроса, кроме токена, необходимо отправить Content-Type, а в теле — JSON с свойствами (все свойства не обязательные):

5. Отображение количества лайков постов

У каждого поста есть свойство likes — оно содержит массив пользователей, лайкнувших пост:

{
  "likes": ["622bd81b06c7d323b8ae4614", "622b6ffc09b12f80f4c10bde"],
  ...другие данные поста
}

Сделайте так, чтобы на каждом посте было написано, сколько у него лайков.

6. Удаления поста

Удаление чего-то, как правило, безвозвратно. Поэтому перед этим действием стоит спросить пользователя, уверен ли он, что хочет удалить пост. Для этого используйте alert() как в задании с котиками (в дальнейшем его можно будет заменить его на popup).

Удаление должно происходить по кнопке удаления, внешний вид и расположение относительно поста вы придумываете сами.

Чтобы удалить поста, отправьте DELETE-запрос:

DELETE https://api.react-learning.ru/v2/:groupId/posts/:postId

Вместо postId в URL нужно подставить параметр _id поста, который нужно удалить. _id каждого поста есть в JSON:

{
  "likes": [],
  "_id": "622bda5606c7d323b8ae4616", — вот он
  ...другие данные поста
}

В итоге, запрос на удаление поста должен выглядеть так:

DELETE https://api.react-learning.ru/v2/:groupId/posts/622bda5606c7d323b8ae4616

7. Постановка и снятие лайка

Чтобы лайкнуть пост, отправьте PUT-запрос:

PUT https://api.react-learning.ru/v2/:groupId/posts/likes/:postId

Чтобы убрать лайк, нужно отправить DELETE-запрос с тем же URL:

DELETE https://api.react-learning.ru/v2/:groupId/posts/likes/:postId

Вместо postId в URL нужно подставить свойство _id соответствующего поста.

В ответе придёт обновлённый JSON с постом. Массив лайков в нём будет уже обновлён.

При постановке и снятии лайка сердечко должно менять цвет, а счётчик лайков увеличиваться или уменьшаться.

Чтобы изменить количество лайков, нужно отправить на сервер запрос с соответствующим методом. Рекомендуем брать количество лайков из ответа сервера, а не из вёрстки. Иначе могут возникнуть ошибки, когда два пользователя одновременно лайкнут один пост.

8. Обновление аватара пользователя (доп задание)

Чтобы сменить аватар, отправьте такой PATCH-запрос:

PATCH https://api.react-learning.ru/v2/:groupId/users/me/avatar

В теле запроса передайте JSON с единственным свойством — avatar. Это свойство должно хранить ссылку на новый аватар. Если отправить не ссылку, сервер вернёт ошибку.

9. Создание и удаление комментарий к постам (доп задание)

Чтобы добавить комментарий пост, отправьте POST-запрос:

POST https://api.react-learning.ru/v2/:groupId/posts/comments/:postId

Чтобы убрать лайк, нужно отправить DELETE-запрос:

DELETE https://api.react-learning.ru/v2/:groupId/posts/comments/:postId/:commentId

Вместо postId в URL нужно подставить свойство _id соответствующего поста.

Вместо commentId в URL нужно подставить свойство _id соответствующего комментария, комментарий должен быть добавлен именно к указанному посту.

В ответе придёт обновлённый JSON с постом. Массив комментариев в нём будет уже обновлён.

При добавлении и удалении комментариев все изменения должны отображаться в DOM.

Так же есть два роута связанные с комментариями:

Чтобы получить список всех комментариев, отправьте GET-запрос:

GET https://api.react-learning.ru/v2/:groupId/posts/comments/

Чтобы получить список комментариев к конкретному посту по его _id, отправьте GET-запрос:

GET https://api.react-learning.ru/v2/:groupId/posts/comments/:postId

4. Постановка и снятие лайка

Чтобы лайкнуть товар, отправьте PUT-запрос:

PUT https://api.react-learning.ru/products/likes/:productId

Чтобы убрать лайк, нужно отправить DELETE-запрос с тем же URL:

DELETE https://api.react-learning.ru/products/likes/:productId

Вместо postId в URL нужно подставить свойство _id соответствующего товара.

В ответе придёт обновлённый JSON с товаром. Массив лайков в нём будет уже обновлён.

При постановке и снятии лайка сердечко должно менять цвет, а счётчик лайков увеличиваться или уменьшаться.

Чтобы изменить количество лайков, нужно отправить на сервер запрос с соответствующим методом. Рекомендуем брать количество лайков из ответа сервера, а не из вёрстки. Иначе могут возникнуть ошибки, когда два пользователя одновременно лайкнут один товар.

Удалять товары может только пользователь с правами администратора.

Поле isAdmin:true в БД.

Удаление должно происходить по кнопке удаления, внешний вид и расположение относительно товара вы придумываете сами.

9. Создание и удаление отзывов к товарам

Чтобы добавить отзыв к товару, отправьте PUT-запрос:

PUT https://api.react-learning.ru/products/review/:productId

Чтобы убрать лайк, нужно отправить DELETE-запрос:

DELETE https://api.react-learning.ru/products/review/:postId/:reviewId

Вместо productId в URL нужно подставить свойство _id соответствующего товара.

Вместо reviewId в URL нужно подставить свойство _id соответствующего отзыва, отзыв должен быть добавлен именно к указанному товару.

В ответе придёт обновлённый JSON с постом. Массив отзывов в нём будет уже обновлён.

При добавлении и удалении отзыва все изменения должны отображаться в DOM.

Так же есть два роута связанные с отзывами:

Чтобы получить список всех отзывов, отправьте GET-запрос:

GET https://api.react-learning.ru/products/review/

Чтобы получить список отзывов к конкретному товару по его _id, отправьте GET-запрос:

GET https://api.react-learning.ru/products/review/:productId
